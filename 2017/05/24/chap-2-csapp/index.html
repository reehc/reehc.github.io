1.Information Storage

Most Computers use blocks of 8 bits, or bytes, as the smallest addressable unit of memory. A machine-level program
views memory as a very large array of bytes, referred to as virtual memory. Every byte of memory is identified by a
unique number, known as its address, and the set of all possible addresses is known as the virtual address space.
Compiler and run-time system partitions this memory into more manageable unit to store the different program objects,
that is, program data, instruction, and control information. Various mechanisms are used to allocate and manage the
storage for different parts of the program. This management is all performed within the virtual address space.

1.1 Hexadecimal Notation

Binary notation is too verbose, while with decimal notation it is tedious to convert to and from bit patterns.
Instead, we write bit patterns as base-16, or hexadecimal numbers. Hexadecimal (or simply "hex") uses digits 'O'
through '9' along with characters 'A' through 'F' to represent 16 possible values.

1.2 Data Sizes

Every computer has a *word size*, indicating the nominal size of pointer data. Since a virtual address is encoded
by such a word, the most important system parameter determined by the word size is the maximum size of the virtual
address space. That is, for a machine with a *w*-bit word size, the virtual addresses can range from 0 to
2<sup>w</sup> - 1, giving the program access to at most 2<sup>w</sup> bytes.

1.3 Addressing and Byte Ordering

For ordering the bytes representing an object, there are two common conventions. The convention where the least
significant byte comes first is referred to as little endian, while the convention where the most significant byte
comes first  is referred to as big endian.
Times when byte ordering becomes an issue:
*Binary data are communicated over a network between different machines.
*Looking at the byte sequences representing integer data.
*Programs are written that circumvent the normal type system.

1.4 Representing Strings

A string in C is encoded by an array of characters terminated by the null (having value O) character. Each
character is represented by some standard encoding, with the most common being the ASCII character code.

1.5 Representing Codes

Different machine types use different and incompatible instructions and encoding. Even identical processors running
different operating systems have differences in their coding conventions and hence are not binary compatible.
Binary code is seldom portable across different combinations of machine and operating system.

1.6 Introduction to Boolean Algebra

The Boolean operations |, &, and ~ operating on bit vectors of length *w* form a *Boolean algebra*, for any integer
*w* > 0.
One useful application of bit vectors is to represent finite sets.

1.7 Bit-Level Operations in C

One common use of bit-level operations is to implement masking operations, where a mask is a bit pattern that
indicates a selected set of bits within a word.

1.8 Logical Operations in C

The logical operations treat any nonzero argument as representing TRUE and argument 0 as representing FALSE.
A second important distinction between the logical operators '&&' and '||' versus their bit-level counterparts
'&'and 'I' is that the logical operators do not evaluate their second argument if the result of the expression can
be determined by evaluating the first argument.

1.9 Shift Operations in C

Generally, machines support two forms of right shift:
*Logical.
*Arithmetic.
